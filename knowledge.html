Foundations & Data Link Layer (Weeks 1-6)

Fundamental data communication models, framing, and error detection.

The OSI vs TCP/IP Models Take a closer look..

Networks operate on stacked logical layers. The OSI Model defines 7 explicit physical and logical boundaries, while TCP/IP compresses them into 4 or 5 pragmatic layers.

  • Application (L7): Network process to application (HTTP, FTP, SMTP).
  • Transport (L4): End-to-end data delivery and reliability (TCP, UDP).
  • Network (L3): Routing packets across disparate networks (IP, Routers).
  • Data Link (L2): Node-to-node framing across a physical wire (MAC, Switches).
  • Physical (L1): Electrical signaling across the raw cable (Hubs, Cables).

Data Encapsulation Flow (Downward)

Examiners frequently test encapsulation sequences: as data passes down the physical stack, each distinct layer physically wraps the payload in its own bespoke header.

Framing & Byte Stuffing Take a closer look..

The Data Link Layer is tasked with identifying where a block of physical bits structurally starts and ends. This is called Framing.

Flag Bytes: Special bit patterns (like 01111110) are injected to definitively mark the start and end of a physical frame.

Byte/Bit Stuffing: If the actual payload data coincidentally contains the Flag Byte pattern, the network will prematurely terminate! To prevent this, the transmitter mathematically injects an "Escape" byte (or stuffs a continuous 0) purely to break the pattern. The receiver dynamically extracts it back out.

Cyclic Redundancy Check (CRC) Take a closer look..

CRC is a robust error-detecting polynomial code specifically engineered to detect accidental changes to raw data.

  • If the generator polynomial is \( G(x) \) of degree \( r \), you mechanically append \( r \) zeroes to the end of the data payload \( D(x) \).
  • You perform binary long division (XOR mathematics) of the shifted data by the generator.
  • The resulting Remainder is explicitly slapped onto the original data and transmitted. If the receiver inherently calculates a remainder of zero, the data is flawlessly intact.

ALOHA Protocols

The original random-access schemes — simple but inefficient.

Pure (Unslotted) ALOHA

  • Transmit any time — no synchronisation needed.
  • A frame is vulnerable for 2×frame-time (your frame + anyone who started just before/after).
  • Max efficiency = 1/(2e) ≈ 18%

Slotted ALOHA

  • Only transmit at the start of a time slot.
  • Vulnerable window is halved — only 1×frame-time.
  • Max efficiency = 1/e ≈ 37% — twice as good.

📊 ALOHA Efficiency Comparison

Pure ALOHA: S = G · e−2G  →  max at G=0.5 → 18%
Slotted ALOHA: S = G · e−G  →  max at G=1 → 37%

🎯 First-Success in Slot N — Formula

P(A succeeds first time in slot k) = (Pfail)k-1 × Psuccess

Where Psuccess for A = pA × (1 − pB) — A transmits AND B stays silent.

Example: pA=0.4, pB=0.3, first success in slot 3:
Psuccess = 0.4 × 0.7 = 0.28  |  Pfail = 0.72
Answer = 0.72² × 0.28 = 0.1452

⏱️ Slot Time (Min Frame Size)

The wire must stay busy long enough to detect a collision at the farthest end.

Slot Time = 2 × dprop
dprop = Distance ÷ Speed
With repeater: add (bits ÷ bps)
Example: 500m wire, speed = 2×10⁸ m/s
d = 500 / 2×10⁸ = 2.5 µs
Slot Time = 2 × 2.5 = 5 µs

WLAN — CSMA/CA & 802.11

Wireless can't detect collisions — so it avoids them instead.

🔇

Why WLAN Cannot Use CD

A transmitting antenna blasts out such a powerful signal locally that it completely drowns out any incoming weaker signal. It is physically impossible to "listen while shouting". Therefore, collision avoidance (CSMA/CA) is used instead.

👻 The Hidden Terminal Problem

Problem: A and B both hear the AP but not each other — they transmit simultaneously and collide at the AP.
Fix: RTS/CTS handshake. A sends RTS → AP shouts CTS to everyone → B hears CTS and stays silent.

⏳ DCF Mode — How BTC Counts Down

  1. Generate random BTC (Back-off Time Counter) value.
  2. Wait for medium to be idle for DIFS duration.
  3. Count down BTC by 1 for every idle slot (slot = 20 ms typical).
  4. If medium goes busy → pause counter, wait for idle again + DIFS, then resume.
  5. When BTC = 0 → transmit!
Time consumed in a free block = DIFS + (ticks_consumed × slotTime)

VLANs — Virtual LANs

Logically slice one physical switch into multiple isolated broadcast domains.

What VLANs Do

  • Split one physical switch into multiple logical broadcast domains.
  • Ports are assigned to a VLAN — traffic stays inside that VLAN.
  • To communicate between VLANs, you still need a Router (or L3 switch).
  • VLANs do NOT reduce collision domains (switches already do that per port).

802.1Q Trunk Tag

  • When a frame travels between switches on a trunk link, a 4-byte tag is inserted.
  • Tag sits between the Src MAC and Type/Length field.
  • Contains: VLAN ID (12 bits) + priority bits.
  • Receiving switch reads tag, strips it, delivers to correct VLAN port.

Switch — MAC Address Table & Aging

Switches learn which device is on which port by inspecting source MAC addresses.

🧠 Switch Self-Learning Logic

1
Frame arrives on port X from MAC "AA"
2
Record AA → Port X in MAC table. Reset TTL to max (e.g. 300s).
3
Look up destination MAC. If found → forward to that port only. If not → flood to all ports.
4
TTL counts down every second. If entry expires → deleted → flood again next time.
⚠️

The TTL Trick (Exam Trap!)

The switch ONLY resets TTL when it receives a frame FROM a device (Source MAC). Sending a frame TO a device has absolutely zero effect on that device's TTL counter. The timer just keeps counting down.

📍 Broadcast MAC Address

Broadcast frames (destined to FF-FF-FF-FF-FF-FF) are forwarded to ALL ports by a switch. This is why switches don't break broadcast domains — only Routers (or VLANs) do.

All-zeros (00-00-00-00-00-00) = null/unassigned — NOT broadcast!
All-ones (FF-FF-FF-FF-FF-FF) = broadcast ✅

Transport Layer & Checksums (Weeks 12-13)

TCP, UDP, and End-to-End Logical Host Reliability.

TCP vs UDP Characteristics Take a closer look..

TCP (Transmission Control Protocol)

  • Connection-Oriented: Enforces a strict 3-way handshake before talking.
  • Reliable: Uses acknowledgements (ACKs) and retransmissions.
  • Congestion Control: Dynamically throttles speed based on network physics.
  • Used for: Web (HTTP), Email (SMTP), File Transfer (FTP).

UDP (User Datagram Protocol)

  • Connectionless: Blasts packets instantly without establishing a connection.
  • Unreliable: Zero guarantee of delivery! No ACKs, No retransmissions.
  • Fast & Barebones: Significantly lower overhead and latency.
  • Used for: Streaming Video, Voice VoIP, Online Gaming.

TCP 3-Way Handshake

Examiner's Reliability Trap

This strict 3-message sequence completely establishes the connection before a single byte of actual application payload data is natively transmitted. This algorithm explicitly isolates TCP as "Connection-Oriented," distinguishing it mechanically from UDP.

Internet Checksum Calculation Take a closer look..

The standard Internet Checksum (used fundamentally in IPv4, TCP, and UDP) is a fast 16-bit One's Complement Summation. It is evaluated via software to provide a secondary backstop against corruption, unlike the vigorous hardware-level CRC math at Layer 2.

  1. Partition the entire data sequence into 16-bit binary segments.
  2. Add them fundamentally together using standard binary addition.
  3. Examiner Trap: If a high-bit carry overflows past 16 bits, it "wraps around" and is immediately added inherently to the rightmost bit of the result (end-around carry).
  4. The final computed Checksum is the One's Complement (bitwise NOT) of the finalized aggregated sum!
  5. Validation: The receiver inherently totals ALL data words AND the checksum. If the packet is flawlessly preserved, the validation sum is universally 1111111111111111.
